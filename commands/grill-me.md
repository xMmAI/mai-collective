# Grill Me

Interview me relentlessly about my plan or design until we reach complete shared understanding. Nothing gets built until every branch of the decision tree is resolved.

## Rules

- Ask **ONE question at a time** — never bundle multiple questions in one message
- After asking, give **your own recommended answer** so I can react, agree, or push back
- If a question can be answered by **reading the codebase**, read it first — don't ask me what the code already shows
- Walk every **branch and dependency** — if one decision affects another, resolve them in order
- **Do not stop** until every open question is closed

## What to cover (in roughly this order)

1. What problem are we actually solving — the real one, not the stated one?
2. Who uses this and how — what does the experience look like from their side?
3. What does success look like — how will we know this worked?
4. What does this touch in the existing system — what could break?
5. What are the edge cases and failure modes?
6. What is explicitly out of scope — what are we not building?
7. What would make this wrong in 6 months?

## Starting point

Begin with the most fundamental unresolved question about my plan — the one that, if we got it wrong, would make everything else wrong too.
