# Zoom Out

I don't know this area of code well. Step back from the current file and give me the bigger picture.

Map the following:

- **All relevant modules** — what each one does in plain English
- **Callers** — which parts of the codebase depend on or call into this area
- **Dependencies** — what this area calls or relies on
- **Data flow** — how data moves through this section from entry to exit
- **Seams** — where the natural boundaries and separation points are

Use plain language. No code unless it's essential for understanding the structure. Name things the way the project names them.
