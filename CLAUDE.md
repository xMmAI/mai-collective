# MAI Collective — Project Instructions

## README auto-update rule

**Whenever a new skill or slash command is added, update README.md immediately following the existing pattern:**

1. **New skill** (`skills/SkillName/SKILL.md` created):
   - Add a description under "Skills: what each one actually does" — plain prose, same length and style as existing entries
   - Add the folder to the folder structure diagram under `skills/`
   - If it adds a slash command, add that to the slash commands table and the relevant "What can MAI be used for?" section

2. **New slash command** (`commands/command-name.md` created):
   - Add a row to the slash commands table (keep alphabetical order within each section)
   - Add it to the relevant "What can MAI be used for?" use case section (developers / writers / product)
   - Update the commands count in the folder structure comment if present

Do not skip this. The README is the public-facing record of what MAI can do.
